# Platform Compliance Checklist

Platform rules change. Before each manual upload, the user must check the current official rules and interface rather than relying on this repository.

## Required operating boundary

- Use the plugin only to create and validate local files.
- Let a human user review and perform every login, upload, topic selection, collection choice, visibility choice, scheduling choice, and publication action.
- Do not automate human-like interaction, reuse sessions, conceal automation, bypass detection, evade rate limits, or work around account restrictions.
- Follow any current platform requirement for labeling or disclosing AI-assisted or generated content.
- Use only images, fonts, quotations, music, trademarks, and other materials that the user has permission to publish.
- Do not fabricate experiences, testimonials, performance figures, sources, identities, or product claims.
- Remove private faces, contact details, precise locations, license plates, account data, analytics, and conversation content unless publication is lawful and intentional.
- Treat health, legal, financial, employment, and other high-stakes claims as requiring qualified review and current authoritative sources.
- Stop if the account is restricted, verification is requested, ownership is unclear, or the user cannot confirm the current rule.

This checklist does not guarantee platform acceptance or account safety.

# Privacy Policy

Rednote Content Kit is designed for local execution.

## Data handling

- The plugin does not transmit data to the publisher.
- The plugin does not include telemetry, analytics, account login, cookies, tokens, passwords, or remote storage.
- The handoff builder reads only the specification and relative image paths explicitly supplied by the user.
- The builder copies selected images into a local output directory and writes a local Markdown handoff plus JSON manifest.
- Output manifests use relative paths and do not preserve source-machine absolute paths.

## User responsibility

Users control any image-generation tool or model they choose to invoke. Those tools may have their own privacy policies. Users must not place credentials, private analytics, personal identifiers, or content without distribution rights in an issue, pull request, example, or public repository.

## Deletion

The plugin has no server-side account or retained cloud data. Delete generated local output files to remove them.

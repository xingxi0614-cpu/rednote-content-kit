# Contributing

## Current status

External code pull requests are not accepted while the patent and license decision is pending. Issues that contain no private data may be used to discuss bugs and documentation.

After the release hold is cleared, contributors must:

1. Work from a clean fork and create a focused branch.
2. Add or update tests for behavior changes.
3. Run the unit tests and release audit locally.
4. Confirm that the contribution contains no credentials, personal information, private platform data, copyrighted images, long third-party quotations, or machine-specific paths.
5. Confirm that they have the right to contribute the work under the repository's selected license.

If commercial dual licensing is selected, a contributor agreement that permits relicensing must be adopted before accepting third-party code. A sign-off alone does not automatically provide the relicensing rights required for that business model.
